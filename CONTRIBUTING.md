# Contributing to Nirog AI

Thank you for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/Nirog-AI.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Set up the project locally — see the [README](README.md#-local-installation) for setup steps

## Making Changes

- Keep changes focused — one feature or fix per pull request
- Follow the existing code style in each layer (ESLint for JS/TS, PEP 8 for Python)
- Test your changes locally before submitting

## Submitting a Pull Request

1. Commit your changes with a clear message: `git commit -m "Add: brief description"`
2. Push to your fork: `git push origin feature/your-feature-name`
3. Open a Pull Request against the `main` branch
4. Describe what the PR does and why

## Reporting Bugs

Open an issue with:
- A clear title
- Steps to reproduce
- What you expected vs what happened
- Your environment (OS, Node.js version, Python version)

## Suggesting Features

Open an issue tagged `enhancement` with a description of the feature and the problem it solves.

## Important

This project deals with health-related information. Please ensure any AI-facing changes include the appropriate medical disclaimer context and do not claim diagnostic accuracy.
